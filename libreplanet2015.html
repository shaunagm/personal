Activity 1: The Setup Sprint

Goals
Maintainers: make your project easier to setup.
Contributors: understand environment setup problems and how to fix them.
Steps
1: Pair up (or small-group-up)
2: Maintainer gets 5 words to describe the project (no URLS!)
3: Contributor tries to find the project and set it up
4: If/when the contributor has difficulty, explain the problem to maintainer
5: Submit issues together to update the documentation and/or install process

Activity 1: The Setup Sprint

Questions:
Where did people get stuck?
What kind of fixes did you come up with?
What problems most benefited from having the maintainer right there?
Tips:
Do this exercise at least once for every operating system you want people to be able to contribute from.
Screen-sharing can help you "look over" a contributor's shoulder.
If people are getting stuck on tools you use - for instance, git - prominently link to that tool's documentation.

Activity 2: User Testing

Goals:
Maintainers: see where users are having trouble with your project.
Contributors: learn more about how you interact with projects.

Activity 2: User Testing

Think Aloud Steps:
1: Practice first by having the tester describe a non-software related task. For instance, they could think through counting the number of windows in their house, or solving a Bongard problem.
2: The tester is practicing thinking aloud. Their partner is practicing asking questions:
- When the tester is silent, ask, "What are you thinking about?"
- If the tester is confused, try to clarify what is confusing. If the tester is frustrated, try to clarify what precisely is causing frustration.
3: Ready? Give the tester a link to the project, or pull it up on maintainers' computer.
4: Maintainer should give the tester a goal such as "sign up on the site", "draw and save a picture", etc.

Activity 2: User Testing

Questions:
What did you learn from this exercise?
Are there particular types of tasks or projects that Think Alouds are well suited for?
Was it hard to limit yourself to clarifying questions?

Activity 3: Roleplaying Interactions

Goals:
Maintainers: get comfortable welcoming new contributors to your project.
Contributors: practice asking questions and stating needs.
Steps:
1: Pair up (or small-group-up) & get some index cards (online versions here)
2: Contributor picks a "character card" who they'll be acting out
3: Maintainer practices responding (use "questions card" if you need help!)
4: If you get through the cards, brainstorm some additional scenarios and try them out.

Activity 3: Roleplaying Interactions

Questions:
What approaches seemed most successful?
What kinds of questions were the hardest to answer?
What new scenarios did you brainstorm?
How would this be different via IRC or mailing list or other online medium?
Tips:
As newcomers to your project get more comfortable and knowledgeable about it, they can help welcome new people. Encourage new members of your community to answer these kinds of questions as a way to contribute.
